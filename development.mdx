<Info>
  **Prérequis**:
  - Node.js version 19 ou supérieure
  - Un dépôt de documentation avec un fichier `docs.json`
</Info>

Suivez ces étapes pour installer et exécuter Mintlify sur votre système d'exploitation.

<Steps>
<Step title="Installer le CLI Mintlify">

```bash
npm i -g mint
```
</Step>

<Step title="Prévisualiser localement">

Naviguez vers le répertoire de documentation où se trouve votre fichier `docs.json`, et exécutez la commande suivante :

```bash
mint dev
```

Une prévisualisation locale de votre documentation sera disponible à `http://localhost:3000`.

</Step>
</Steps>

## Ports personnalisés

Par défaut, Mintlify utilise le port 3000. Vous pouvez personnaliser le port sur lequel Mintlify s'exécute en utilisant le flag `--port`. Par exemple, pour exécuter Mintlify sur le port 3333, utilisez cette commande :

```bash
mint dev --port 3333
```

Si vous tentez d'exécuter Mintlify sur un port déjà utilisé, il utilisera le prochain port disponible :

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Validation des liens

Le CLI peut aider à valider les liens dans votre documentation. Pour identifier les liens cassés, utilisez la commande suivante :

```bash
mint broken-links
```

## Formatage du code

Nous suggérons d'utiliser des extensions sur votre IDE pour reconnaître et formater le MDX. Si vous êtes un utilisateur VSCode, considérez l'[extension MDX VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) pour la coloration syntaxique, et [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) pour le formatage du code.

## Dépannage

<AccordionGroup>
  <Accordion title='Erreur: Could not load the "sharp" module using the darwin-arm64 runtime'>

    Cela peut être dû à une version obsolète de Node.js. Essayez ce qui suit :
    1. Supprimez la version actuellement installée du CLI : `npm remove -g mint`
    2. Mettez à jour vers Node.js v19 ou supérieur.
    3. Réinstallez le CLI : `npm i -g mint`
  </Accordion>

  <Accordion title="Problème: Rencontre d'une erreur inconnue">
  
    Solution : Allez à la racine de votre appareil et supprimez le dossier `~/.mintlify`. Ensuite, exécutez `mint dev` à nouveau.
  </Accordion>
</AccordionGroup>
